# Propagate CLI Command Reference

This reference covers the current CLI commands implemented under `packages/cli/internal/cli`.

Use `--help` on the root command, command groups, or leaf commands when usage is unclear. Use `--json` for machine-readable output, `--non-interactive` to fail instead of prompting, `--no-color` for plain terminal output, and `--api-url VALUE` when the Propagate API URL is not available from local configuration.

## Command Matrix

Root aliases are supported for help and version output: `propagate --help`, `propagate -h`, `propagate --version`, and `propagate -v`. Command groups also accept `help`, `--help`, or `-h`.

## Safety Flags

`--dry-run` previews planned local or cloud changes without writing.

`--yes` confirms a write-capable operation after review. It does not provide env values and does not bypass explicit join decisions.

`--non-interactive` refuses prompts. If a prompt would be needed and the required flags are missing, the command exits with a confirmation-required error.

`--value-stdin` is only for `propagate env set`. It reads exactly one single-line value from stdin and keeps positional env values rejected.

`propagate env set` uses the only configured scope automatically. When multiple scopes exist, humans get a scope picker and non-interactive runs must pass `--scope`.

`propagate run` has no dry-run mode because it does not write env files or cloud values. It refreshes `propagate.yaml` from the cloud before starting the child process, and requires confirmation or `--yes` before overwriting local config changes. Use `--yes` for approved prod injection, and remember that child process output is not sanitized by Propagate.

## Agent Rules

Prefer `propagate status` for broad discovery and focused status commands for one area. Prefer `--json` for parsing and summaries. For write-capable commands, show the dry-run summary first, then ask for human approval before using `--yes` or before running an interactive command.

Never write env values to docs, prompts, fixtures, commits, issue text, pull request text, or agent instructions.

Use `propagate run --scope dev -- COMMAND [args...]` only when the command itself needs the values. Do not use it to inspect, print, or capture secrets.

When a command reports a permission problem, stale revision, missing identity, missing `propagate.yaml`, or pending join requirement, stop and report the next safe command from the CLI output.
